  • 에러가 발생한다면 아래 체크리스트를 점검한다.

(1) ApplicationDataType과 ImplementationDatyaType의 data mapping을 수행했는가?

(2) ApplicationDataType과 ImplementationDatyaType Category는 동일한가?(혹은 Mapping Category Table을 만족하는가?)

(3) 참조하는 DataConstrs의 DataConstrs Rule에 Phys Constrs가 존재하는가?

  • ADT_uint8 [/Common/ApplicationDataTypes/ADT_uint8]

  • ApplicationPrimitiveDatatype 생성 (예시 : uint8)

  • DataTypeMappingSet0 [/Common/DataTypeMappings/DataTypeMappingSet0]

  • Data Mapping 수행
    • Data Type Mapping Set 생성 후, New 클릭
    • 생성한 ApplicationDataType과 알맞은 ImplementationDataType을 연결한다.
  • 단일 자료형인 ApplicationPrimitiveDataType의 경우 AUTOSAR에 기본 제공된 ImplemenationDataType을 사용한다.
  • /AUTOSAR_DataTypes/ImplementationDataTypes/AUTOSAR_uint8
3
  • ADT_Structure [/Common/ApplicationDataTypes/ADT_Structure]

  • ApplicationRecordDataType 생성
    • New 버튼으로 Element를 생성한다.
    • Type : ApplicationPrimitiveDataType 중 원하는 것으로 설정
    • Category : ApplicationPrimitiveDataType과 동일한 것으로 설정

4
  • IDT_Structure [/Common/ImplementationDataTypes/IDT_Structure]

  • IDT_Structure > Sub Element

  • ImplementationDataType 생성
    • 생성한 ApplicationRecordDataType에 대응하는 ImplementationDataType을 생성해야한다.
    • Category : STRUCTURE
    • Sub Elements 생성 및 설정
      • Sub Elements 생성 시 ApplicationRecordDataType의 Elements와 동일한 category 
      • Sub Elements 생성 시 알맞는 Base Type을 설정해야한다. (AUTOSAR 제공 Base Type 사용)
  • 개요의 ImplementationDataType 조건 참고
    • ApplicationDataType, ImplementationDatyaType Category 동일
    • ImplementationDataType의 Sub Elements는 ApplicationRecordDataType의 Elements와 동일
5
  • DataTypeMappingSet0 [/Common/DataTypeMappings/DataTypeMappingSet0]

  • Data Mapping 수행
    • Data Type Mapping Set 생성 후, New 클릭
    • 생성한 ApplicationDataType과 알맞은 ImplementationDataType을 연결한다.

6
  • If_SearSwitch_Structure [/App_Rte/If_SearSwitch_Structure]

Sender와 Receiver의 All Contents > Internal Behaviors > Data Type Mappings 세팅


  • 사용예제
    • Sender-Receiver Interface로 사용
    • 연결된 PPort와 RPort가 같은 Type인지 확인
    • Port 탭, Runnable 탭 내 설정 완료 후 Sender와 Receiver의 All Contents > Internal Behaviors 에서 Target Data가 올바르게 설정되었는지 확인

4. Appendix

4.1 ApplicationPrimitiveDataType

  • 기본형 타입
  • 아래의 ApplicationCompositeDataTypes 생성하기 위해서는 알맞는 ApplicationPrimitiveDataType을 먼저 생성해야한다.

4.2  ApplicationCompositeDataTypes 

  • C언어에서의 구조체 혹은 데이터 배열과 유사하게 정의된 Application Data Level으로 다음 2가지가 있다. 

4.2.1 ApplicationRecordDataType (구조체)

- ApplicationRecordDataType을 구성하는 ApplicationRecordDataElement(구조체의 멤버)는 다음 Category를 따를 수 있다. : VALUE, VAL_BLK, STRUCTURE, ARRAY, STRING, BOOLEAN, CURVE, MAP

- ApplicationRecordDataType의 경우는 Category가 반드시 STRUCTURE로 설정되어야한다.

4.2.2 ApplicationArrayDataType (배열)

- ApplicationArrayDataType은 동일한 ApplicationDataType의 하나 이상의 ApplicationArrayElement로 구성된다. (array를 이루는 element들의 data type이 동일해야함)

- arraySizeSemantics에 따라 element 수는 0 ~ maxNumberOfElements 사이로 달라진다.

- 해당 카테고리는 참조되는 AutosarDataType의 Category와 동일해야한다 : VALUE, VAL_BLK, STRUCTURE, ARRAY, STRING, BOOL, COM_AXIS, RES_AXIS, CURVE, MAP

4.3 ImplementationDatyaType 

  • ApplicationDataType은 반드시 ImplementationDatyaType에 mapping 되어야한다. 

→ ApplicationDataType은 Application Value Constant를 표현하고, ImplementationDataType은 Numerical Value을 의미함. 따라서 두 타입 간 별도의 상수 처리가 필요하게 됨 = Data Mapping

  • 이를 위한 ImplementationDatyaType은 다음을 만족한다.

- ApplicationPrimitiveDataType과 Mapping 시 아래 Mapping Category Table을 만족

- ApplicationCompositeDataType과 Mapping 시 동일한 Category

- 동일한 구조 (ImplementationDataTypeElement와 해당 subElements를 참조한다.)

- arraySize와 arraySizeSemantics에 대해 동일한 value

- Data types에 연결된 호환 가능한 swDataDefProps
